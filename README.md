# **AIC JBOD Disk Power Controller Tool (磁碟機櫃電源梯次控制工具)**

這是一個專為 **AIC JBOD (儲存磁碟機櫃)** 設計的 Windows 自動化運維腳本。其核心功能在於提供「安全、漸進式（Staggered）」的硬碟批次開機機制，以及快速的批次關機功能，有效避免大量硬碟同時啟動時所產生的瞬間高電流（突波）導致機房供電系統過載。

## **核心設計原理**

### **1\. 梯次開機與 OS 智能偵測 (Power ON)**

當執行批次開機時，工具不會一口氣將所有硬碟通電，而是採用 **「點亮一顆 $\\rightarrow$ 確認認到 $\\rightarrow$ 再點亮下一顆」** 的安全循環：

* **基准線建立：** 透過 Windows 內建的 diskpart 統計目前系統已存在的實體磁碟總數。  
* **逐顆發電：** 透過 SES 指令點亮下一個硬碟插槽。  
* **動態輪詢 (Poll Loop)：** 每隔 3 秒利用 diskpart 重新計數。當發現「當前磁碟數 $\>$ 初始磁碟數」（即 Windows 成功辨識到新硬碟）時，才會推進到下一顆硬碟。  
* **超時保護：** 若單顆硬碟通電超過 180 秒仍未被 OS 辨識，腳本將暫停並跳出警告，由管理員決定跳過或中止。

### **2\. 快速批次關機 (Power OFF)**

關機時不需等待作業系統回應，腳本會依序向各磁碟插槽發送斷電指令，每顆硬碟之間固定**延遲 1 秒鐘**，以規整、快速的方式完成機櫃關機。

## **環境與相容性需求**

### **1\. 支援作業系統**

* Windows Server 2012 / 2016 / 2019 / 2022 或更新版本  
* Windows 10 / 11

### **2\. 外部相依工具 (Dependencies)**

本腳本依賴 **sg3\_utils** 套件中的兩個核心工具。請確保這兩個檔案與腳本放置在**同一個資料夾**下，或者已加入系統的 PATH 環境變數中：

* sg\_scan.exe：用於掃描與定位 AIC 擴展卡與機櫃內部控制器。  
* sg\_ses.exe：用於發送 SES (SCSI Enclosure Services) 指令來控制插槽電源。

## **使用操作指南**

### **步驟 1：以系統管理員身分執行**

由於腳本需要呼叫 diskpart 讀取硬體變更，並透過 sg\_ses 操控硬體暫存器，**必須使用系統管理員權限執行**：

1. 右鍵點擊 AIC\_Disk\_Power\_Controller.bat。  
2. 選擇 **「以系統管理員身分執行」**。

### **步驟 2：選單功能操作**

執行後會出現主選單介面：

Plaintext  
\===================================================  
            AIC JBOD Disk Power Controller Tool  
\===================================================  
 \[1\] Power ON  Disks (Staggered, wait for OS detection)  
 \[2\] Power OFF Disks (Fast shutdown, 1s delay)  
 \[3\] Exit  
\===================================================  
Select action (1-3): 

* **輸入 1 (開機)：** 程式將啟動自動偵測（優先尋找 P0 裝置），利用正則表達式 /R /C:"descriptor: Disk\[0-9\]" 排除 DiskPowerSupply 等干擾項，精準計算出實際硬碟槽總數。確認資訊無誤後輸入 y 即可開始安全梯次開機。  
* **輸入 2 (關機)：**  
  確認提示後，將會以 1 秒的間隔依序安全關閉機櫃所有硬碟。  
* **輸入 3 (離開)：**  
  結束並關閉工具。

## **維護與故障排除 (Troubleshooting)**

### **Q1：畫面出現 : was unexpected at this time. 崩潰訊息？**

* **原因：** 通常是因為複製貼上程式碼時，不小心混入了網頁格式的隱形字元（例如不換行空格 0xA0），導致 Windows Batch 的 if 或 for 語法解析失敗。  
* **解法：** 目前版本已將 :POLL\_LOOP 優化為「單行反向邏輯判斷（消滅大括號大作戰）」，已從架構上完全免疫此問題。若在其他地方遇到，請檢查是否有特殊字元或全形空格。

### **Q2：偵測到的 Disk Count 數量不正確？**

* **原因：** 不同型號的 AIC 機櫃，其內部元件描述文字可能有所變動。  
* **解法：** 可以在 CMD 手動輸入 sg\_ses.exe \-p ed \<你的SCSI代號\> 觀察其輸出格式，並適度修改腳本中 findstr /R /C:"... " 的過濾關鍵字。
